# Вотчер — эксплуатация и чтение логов

Короткая методичка: как запускать, обновлять данные и понимать, что происходит.

## 1. Запуск

- **Первый раз** (один раз): `python -m venv .venv` → `.venv\Scripts\activate` → `pip install -r requirements-dev.txt` → `copy .env.example .env` (в `.env` поставить `DATABASE_URL=sqlite:///db.sqlite3`) → `python manage.py migrate` → `python manage.py createsuperuser`.
- **Каждый день**: `.venv\Scripts\activate` → `python manage.py runserver` (или двойной клик по `run.bat`).
- **Правило**: любую команду `python manage.py ...` запускать только после `.venv\Scripts\activate`. Без активации возьмётся системный Python без зависимостей — будет `ModuleNotFoundError: No module named 'celery'`. Это не поломка проекта, а просто не то окружение.

## 2. Данные: демо и реальные не смешиваются

- `python manage.py seed_demo` — кладёт 4 демонстрационных фильма (чтобы UI и график выглядели наглядно без интернета). Демо-сеансы помечены `source="demo"`, и в таблице «Где смотреть» у них стоит серый бейдж **демо**.
- `python manage.py scrape` — реальный сбор с Афиши и Москино (нужен интернет). Реальные сеансы получают `source="afisha"`/`"moskino"` и кнопку **Билеты** (ссылку на страницу покупки).
- `run.bat` больше **не** запускает `seed_demo` сам — поэтому демо не подмешивается к реальным данным.
- **Перед защитой** выбери одно состояние базы:
  - только демо (стабильно, без интернета), или
  - только реальные: `python manage.py flush` (очистить) → `python manage.py scrape`.
  Смешивать не нужно — иначе на странице рядом будут реальные билеты и демо-строки.

## 3. Обновить цены (реальный сбор)

```
python manage.py scrape
```

Опрашивает все источники, по каждому пишет строку в журнал и печатает в консоль, сколько собрано или какая ошибка. Один источник упал — остальные всё равно соберутся.

## 4. Чтение логов сбора — самое важное

Журнал прогонов: **админка → Сбор цен → «Запуски парсеров»** (`/admin/scraping/scraperun/`).

В каждой строке: источник, статус, сколько сеансов собрано, текст ошибки, время.

Статусы:

- **success** — собрано N сеансов.
- **error** — прогон упал, смотри поле «Ошибка».

Типичные ошибки и что делать:

- `Server disconnected` / `WinError 10054` — сервер источника сбросил соединение (антибот или сетевой сбой). Парсер теперь сам делает 2 повтора с паузой; если всё равно error — просто запусти `scrape` позже.
- `ParserBlocked: HTTP 403/429/503` — источник заблокировал (слишком частые запросы). Увеличь `SCRAPER_DELAY_SECONDS` в `.env`.
- `Получено 0 сеансов...` — на странице источника не нашлось сеансов: либо изменилась вёрстка, либо на эти даты показов нет.

Честная фраза для защиты: «основной рабочий источник — Афиша; Москино подключён как дополнительный, внешние сайты иногда рвут соединение, поэтому каждый прогон фиксируется в журнале со статусом».

## 5. Какие кинотеатры собираются

- **Афиша** опрашивает кинотеатры с галочкой «Мониторим» (админка → Кинотеатры). По умолчанию — «Иллюзион» и «КАРО 11 Октябрь». Задача `discover_cinemas` дописывает новые площадки Москвы; ставишь галочку — со следующего `scrape` они собираются.
- **Москино** — вся городская сеть (~20 кинотеатров) одной страницей.

## 6. Telegram-уведомления (опционально)

- Включается токеном в `.env` (`TELEGRAM_BOT_TOKEN`, `TELEGRAM_BOT_USERNAME`). Без токена просто выключено, сервис работает.
- Проверка: «Мои подписки» → «Привязать Telegram» → нажать Start у бота → привязка в течение минуты. Уведомление приходит, когда цена опускается ниже порога подписки.
- Журнал отправленных: админка → Уведомления.

## 7. Проверки качества

```
pytest            (30 тестов)
ruff check .      (стиль)
python manage.py check
```

## 8. Частые вопросы

- «Показывает только 4 фильма» — это демо-данные; запусти `python manage.py scrape` для реальных.
- «manage.py падает на celery» — не активирован `.venv`.
- «invalid literal for int()» — в `.env` пустая числовая переменная (порча при копировании); впиши `SCRAPE_INTERVAL_MINUTES=240`.
