# Вотчер — план разработки (статус на 10.06.2026)

Выпускной проект цифровой кафедры. Срок: 14 дней. Город: Москва.
Код: `kinomonitor/`. Деплой: `docs/deploy.md`.

## 1. Стек (зафиксирован)

Python 3.12 · Django 5.2 + DRF · Celery + Redis (beat) · PostgreSQL 16 ·
httpx + BeautifulSoup4 (lxml) · HTMX + Chart.js + Bootstrap 5 ·
Telegram Bot API · Docker Compose · pytest + ruff.

## 2. Архитектура

Монолит из четырёх приложений: **catalog** (кинотеатры, фильмы, сеансы, цены,
страницы и API), **scraping** (парсеры-адаптеры, Celery-задачи, журнал ScrapeRun),
**accounts** (регистрация, подписки с порогом цены), **notifications**
(Telegram, журнал отправок — защита от дублей).

Паттерн парсер-адаптер: источник = класс с `fetch_sessions() -> list[SessionDTO]`,
регистрация через `__init_subclass__`. Пайплайн: fetch → нормализация названий →
upsert → снимок цены (пишется только при изменении). Beat: сбор и проверка
порогов каждые 4 часа; каждый источник — отдельная задача с ретраями.

через Афишу, поэтому отдельный парсер сайта не нужен.
Этика: интервал 4 ч, User-Agent с пометкой учебного проекта, точечные страницы.

## 5. Что осталось сделать руками

1. **Деплой** — нужен VPS (вопрос к кафедре/свой). Чем раньше, тем длиннее история цен к защите.
2. **Боевой запуск парсеров** — фикстуры сверены с живыми страницами 10.06.2026,
   но первый запуск проверить по журналу «Запуски парсеров»; разбор не привязан
   к CSS-классам, при смене вёрстки чинить `parse_*` функции по новым фикстурам.
3. **Пояснительная записка по ГОСТ** — если кафедра требует (открытый вопрос).
4. Скриншоты живого сервиса в презентацию (слайд «Демонстрация»).

## 6. Риски

Антибот Афиши при частых запросах → интервал 4 ч и 2 кинотеатра, при блоке —
Москино остаётся полностью рабочим источником. Смена вёрстки → тесты на
фикстурах + журнал падений. Мало истории к защите → деплой раньше, beat каждые 4 ч.
