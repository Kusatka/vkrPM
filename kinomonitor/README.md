# Вотчер

Сервис мониторинга цен и расписания сеансов в кинотеатрах Москвы. Выпускной проект цифровой кафедры.

Django 5 + DRF · Celery + Redis · PostgreSQL · парсинг httpx + BeautifulSoup · HTMX + Chart.js · Docker.
Стек: [`../docs/stack.md`](../docs/stack.md) · план: [`../docs/plan.md`](../docs/plan.md) · деплой: [`../docs/deploy.md`](../docs/deploy.md) · отчёт: [`../docs/`](../docs/).

## Быстрый старт (первый раз, один раз)

```bat
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements-dev.txt
copy .env.example .env
```

Для локального запуска без Docker поставьте в `.env` строку базы на SQLite:
`DATABASE_URL=sqlite:///db.sqlite3`. Затем:

```bat
python manage.py migrate
python manage.py createsuperuser
python manage.py seed_demo
python manage.py runserver
```

Открыть: <http://localhost:8000> — афиша, `/new/` — новинки, `/admin` — админка, `/api/` — REST API.

## Ежедневный запуск

`venv` и зависимости ставятся один раз. Дальше просто:

```bat
.venv\Scripts\activate
python manage.py runserver
```

или двойной клик по **`run.bat`** — он сам сделает `migrate` + `runserver`. `migrate` безопасен: ничего не делает, если миграции уже применены. Демо-данные больше **не** подсыпаются автоматически (чтобы не смешивать их с реальными). Наполнить базу: **реальные** фильмы — `python manage.py scrape` (или `scrape.bat`), демо для офлайна — `python manage.py seed_demo`.

**Когда нужен `migrate`:** только при первом запуске (пустая БД) и после изменения моделей (появились новые файлы миграций). В обычной работе — не нужен.

## Обновление данных

- **Реальные фильмы, кинотеатры и цены** — основной способ наполнить базу (нужен интернет):

  ```bat
  python manage.py scrape
  ```

  Команда опрашивает Афишу и Москино (можно один источник: `scrape moskino`), пишет каждый прогон в журнал и не падает целиком, если один источник недоступен. После неё на сайте появляются **десятки реальных фильмов** и кинотеатров — это и есть ответ на «почему всего 4 фильма» (4 — это демо-данные). Двойной клик — **`scrape.bat`**.
- **Демо-данные** (4 фильма для офлайн-просмотра интерфейса): `python manage.py seed_demo` (повторно пропустит; пересоздать — `seed_demo --force`).

В Docker сбор идёт сам каждые 4 часа (Celery beat), `discover_cinemas` дописывает новые кинотеатры. Результат любого прогона — в админке: **Сбор цен → Запуски парсеров**.

## Источники данных

| Парсер | Что собирает |
|---|---|
| `afisha` | «Иллюзион» и «КАРО 11 Октябрь» со страниц кинотеатров на Афише.ру: цены «от N ₽», пометка SUB = показ в оригинале |
| `moskino` | Городская сеть «Москино» (~20 кинотеатров) с mos-kino.ru/schedule: точные цены, пометка СУБ |
| `demo` | Генератор данных для разработки UI; не регистрируется по умолчанию |

Список кинотеатров для `afisha` живёт в БД (админка → Кинотеатры → галочка «Мониторим»). Разбор намеренно идёт по тексту страниц, а не по CSS-классам. Подробно: [`../docs/parsing.md`](../docs/parsing.md).

## Возможности интерфейса

- **Афиша** карточками с фильтрами «только в оригинале» и «спецпоказы» (HTMX, без перезагрузки) и **поиском** по названию.
- **Новинки** (`/new/`) — фильмы с предстоящими сеансами, новые по году сверху.
- **Страница фильма** — сравнение цен по кинотеатрам и **график истории цены по кинотеатрам**: каждая линия — один кинотеатр (нижняя = где дешевле), легенда кликабельна. Плюс совет «когда покупать» по тренду цены. В таблице «Где смотреть» у каждого сеанса — кнопка **«Билеты»** на страницу источника для покупки.
- Личный кабинет: подписки на фильм/кинотеатр с порогом цены и привязка Telegram.

## Telegram-уведомления

1. Создать бота у @BotFather; в `.env` указать `TELEGRAM_BOT_TOKEN` и `TELEGRAM_BOT_USERNAME`.
2. На странице «Мои подписки» нажать «Привязать Telegram» — после Start привязка происходит автоматически (задача `poll_telegram_links`).
3. `check_price_drops` шлёт сообщение при цене ниже порога; дубли отсекаются по `NotificationLog`.

Без токена сервис работает — уведомления просто отключены.

## Настройка сбора

`SCRAPE_INTERVAL_MINUTES` (по умолчанию 240) и `SCRAPER_DELAY_SECONDS` (пауза между запросами, 1.0) в `.env`. Спецпоказы (опера, TheatreHD, экскурсии) распознаются по названию и скрыты из афиши по умолчанию.

## Тесты и линтер

```bat
set DATABASE_URL=sqlite:///test.sqlite3
pytest
ruff check .
```

30 тестов: парсеры и discovery на фикстурах, подписки, уведомления, аналитика, актуальная min-цена, пустой прогон = ошибка.

## Как добавить новый источник цен

1. Класс в `apps/scraping/parsers/<источник>.py`, наследник `BaseParser`, реализовать `fetch_sessions() -> list[SessionDTO]`.
2. Добавить импорт модуля в `apps/scraping/parsers/__init__.py`.
3. Сохранить HTML страницы в `tests/fixtures/` и написать тест разбора.

Реестр подхватит класс автоматически, beat начнёт запускать его каждые 4 часа.

## Известные ограничения

- Парсеры читают серверный HTML и **не исполняют JavaScript**; при переходе источника на клиентский рендеринг понадобится headless-браузер.
- Прогон, собравший 0 сеансов, помечается в журнале как ошибка (вероятна смена вёрстки или блокировка). Коды 403/429/503 поднимают `ParserBlocked` и уходят в повтор.
